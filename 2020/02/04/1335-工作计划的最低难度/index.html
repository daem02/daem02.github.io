1335. 工作计划的最低难度

发表于 2020-02-04 | 分类于 LeetCode | 阅读次数：
本文字数： 2.5k | 阅读时长 ≈ 4 分钟

Question

你需要制定一份 d 天的工作计划表。工作之间存在依赖，要想执行第 i 项工作，你必须完成全部 j 项工作（ 0 <= j < i）。

你每天至少需要完成一项任务。工作计划的总难度是这 d 天每一天的难度之和，而一天的工作难度是当天应该完成工作的最大难度。

给你一个整数数组 jobDifficulty 和一个整数 d，分别代表工作难度和需要计划的天数。第 i 项工作的难度是 jobDifficulty[i]。
返回整个工作计划的 最小难度 。如果无法制定工作计划，则返回 -1 。

示例1：

Solution

动态规划方法：
dp[i][j]表示前i+1项工作，在j+1天内刚好完工的最小难度

dp[i][j] = dp[k][j-1] + max(jobDifficulty[k+1:i+1]) 其中(j-1<=k<=i-1)

即在j+1天完成i+1项工作，则可以通过在j天内完成k+1项工作，然后在1天内完成i-k项工作，为保证
前j天内满足要求，则工作量至少为j项。

动态规划找相似子问题（子结构），子结构存在无后向性（无论何种方法到达一个状态，后续的状态都可
以在前面的状态下求解出来），所以dp数组要将变化的状态都保存进（想想背包问题，既要有剩余的物品，
也要保存背包的状态，如果不保存背包状态，由于其是一个变量，后续无法构建重复子问题）。

Pyhton:

递归+回溯方法：
第一天完成k项工作(1<=k<=len(jobDifficulty) - d),剩下的d-1天完成,计算所有的k情况，
选择其中工作量最小的一种。其中剩下d-1天的最小工作量计算方式即为重复子问题，用递归计算。

Python:

遗憾的是这个方法超时了，即使是在使用了lru_cache和改进了tmpmax的计算之后。
